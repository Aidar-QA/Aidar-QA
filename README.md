## <a name="stek" /> ⚙️ Навыки и технологии

### 🔍 Тестирование API и Frontend
![Postman](https://img.shields.io/badge/Postman-FF6C37?style=for-the-badge&logo=postman&logoColor=white) ![Swagger](https://img.shields.io/badge/-Swagger-%2385EA2D?style=for-the-badge&logo=swagger&logoColor=black) ![JWT](https://img.shields.io/badge/JWT-black?style=for-the-badge&logo=JSON%20web%20tokens) ![Charles Proxy](https://img.shields.io/badge/Charles-4298B8?style=for-the-badge&logo=charlesproxy&logoColor=white)

* **API Testing:** Тестирование REST-запросов через **Postman**, httpx и Swagger.
* **Network Debugging:** Анализ и подмена трафика (Rewrite, Breakpoints) с помощью Charles Proxy.
* **SSO/Auth:** Тестирование авторизации (**JWT, OAuth 2.0**, RS256, session management).
* **Frontend:** Кроссбраузерное тестирование и проверка доступности (**Lighthouse, axe DevTools**).

### 🗄️ Работа с данными (SQL)
![SQL](https://img.shields.io/badge/SQL-005C84?style=for-the-badge&logo=sqlite&logoColor=white) ![PostgreSQL](https://img.shields.io/badge/postgres-%23316192.svg?style=for-the-badge&logo=postgresql&logoColor=white) ![MySQL](https://img.shields.io/badge/mysql-%2300f.svg?style=for-the-badge&logo=mysql&logoColor=white)

* **Базы данных:** Уверенное владение **PostgreSQL** и MySQL.
* **Сложные запросы:** Написание запросов с использованием **JOIN**, группировки (**GROUP BY, HAVING**), агрегатных функций и подзапросов.
* **Валидация:** Проверка консистентности данных на уровне БД при проведении интеграционных тестов.

### 🏗️ Инфраструктура и Observability
![Docker](https://img.shields.io/badge/docker-%230db7ed.svg?style=for-the-badge&logo=docker&logoColor=white) ![Kubernetes](https://img.shields.io/badge/kubernetes-%23326ce5.svg?style=for-the-badge&logo=kubernetes&logoColor=white) ![RabbitMQ](https://img.shields.io/badge/RabbitMQ-FF6600?style=for-the-badge&logo=rabbitmq&logoColor=white) ![Apache Kafka](https://img.shields.io/badge/Kafka-black?style=for-the-badge&logo=apachekafka) ![Grafana](https://img.shields.io/badge/Grafana-F46800?style=for-the-badge&logo=grafana&logoColor=white)

* **Message Brokers:** Работа с очередями сообщений (**RabbitMQ, Kafka**) — тестирование асинхронного взаимодействия микросервисов.
* **Мониторинг и логи:** Использование **Grafana** для визуализации, **Loki** для работы с логами и **Tempo** для распределенной трассировки (tracing).
* **Контейнеризация:** Работа с **Docker**, управление подами в **Kubernetes (K8s)**.
* **CI/CD & Tools:** Работа в Git, Jira, GitLab и ведение документации в Confluence.

### 🤖 Автоматизация тестирования
![Python](https://img.shields.io/badge/python-3670A0?style=for-the-badge&logo=python&logoColor=ffdd54) ![Pytest](https://img.shields.io/badge/pytest-%23C93739.svg?style=for-the-badge&logo=pytest&logoColor=white) ![Playwright](https://img.shields.io/badge/playwright-%232EAD33.svg?style=for-the-badge&logo=playwright&logoColor=white) ![Allure](https://img.shields.io/badge/allure-%232EAD33.svg?style=for-the-badge&logo=allure&logoColor=white)

* **Фреймворки:** Pytest, Pydantic, Allure.
* **Инструменты:** **Playwright** для создания надежных E2E и интеграционных сценариев на Python.

# <a name="pet-projects" /> Pet-проекты

## <a name="microservice" /> 📊 Тестирование микросервисов маркетплейса. Сквозное отслеживание заказа через RabbitMQ, Kafka и Grafana.

* Проект: Исследование распределенного отслеживания (Observability) в микросервисной архитектуре.
* Стек: RabbitMQ, Kafka, Grafana, Loki, Tempo.
* Для локального развертывания всей инфраструктуры используется Docker Compose. Это позволяет быстро поднять изолированную среду со всеми необходимыми зависимостями (16 контейнеров).
* Тестовый стенд — [маркетплейс.zip](https://drive.google.com/file/d/1MfKzkF4m8akE3-M4gk61F2nb6gCuyMTj/view?usp=drive_link)
* Для запуска буду использовать pycharm вставляем папку
* Запускаем локально через docker пишем ```docker compose up -d ```
